# pdf2md — PDF를 구조화 마크다운으로 변환

본 스킬은 PDF 한 파일을 원문 손실 없이 구조화된 마크다운으로 변환하는 절차를 정의한다. 50페이지 단위로 서브에이전트(opus 모델)를 병렬 할당하고, 각 에이전트는 담당 페이지 추출물을 Read 도구로 직접 읽어 변환하며, 이미지는 별도 파일로 추출하여 링크로 참조한다.

## 스킬 구조

본 스킬은 **오케스트레이터**, **호출 어시스턴트(Dispatcher)**, **검증 에이전트(Validator)**, **서브에이전트** 네 주체의 책임을 명확히 분리한다.

- **3절 오케스트레이터 작업 범위**: 스킬 실행자(사용자와 대화하는 주체)가 수행하는 작업. 루트 준비, 분해, 큐 관리, 최종 경로 배치, 정리, 사용자 보고. 변환과 검증은 각각 호출 어시스턴트와 검증 에이전트에 위임한다.
- **3.7절 호출 어시스턴트(Dispatcher)**: 오케스트레이터로부터 task.json 목록을 받아, 4절 원본을 직접 Read하여 프롬프트를 조립하고 서브에이전트를 병렬 호출한 뒤 결과를 오케스트레이터에 반환하는 중간 에이전트.
- **3.8절 검증 에이전트(Validator)**: 오케스트레이터로부터 완료된 파트 목록과 메타데이터를 받아, 병합·markdownlint·헤딩 검증·이미지 링크 해소·오탈자 검사를 수행하고 결과를 오케스트레이터에 반환하는 에이전트.
- **4절 서브에이전트 지시문 원본**: 호출 어시스턴트가 Read로 직접 읽어 프롬프트를 조립하는 원본. 오케스트레이터는 이 섹션을 복사하지 않는다.

## 1. 용어

- **오케스트레이터(Orchestrator)**: 본 스킬을 실행하는 주체이자 사용자의 협력자. 사용자의 요청을 받아 큐 구성, 호출 어시스턴트 위임, 병합, 이미지 집계·링크 재작성, 검증, 정리, 보고를 담당한다.
- **호출 어시스턴트(Dispatcher)**: 오케스트레이터가 Agent 도구로 호출하는 중간 에이전트. task.json 목록을 받아 SKILL.md 4절을 직접 Read하여 프롬프트를 조립하고, 서브에이전트를 병렬 호출한 뒤, 완료 보고를 취합하여 오케스트레이터에 반환한다.
- **검증 에이전트(Validator)**: 오케스트레이터가 Agent 도구로 호출하는 에이전트. 완료된 파트를 병합하고, markdownlint·헤딩 검증·이미지 링크 해소·오탈자 검사를 수행한 뒤, 통과/실패 결과와 수정 내역을 오케스트레이터에 반환한다.
- **서브에이전트(Sub-agent)**: 호출 어시스턴트가 Agent 도구로 호출하는 opus 변환 에이전트. 담당 페이지 구간 1개만 변환하며 사용자와 직접 대화하지 않는다. 결과·오류는 호출 어시스턴트에게만 전달한다.
- **part_source**: 서브에이전트가 Read 도구로 직독하는, 담당 페이지만 포함된 PDF 추출물. 오케스트레이터가 분해 시 qpdf로 사전 생성한다.
- **조건 플래그**: 오케스트레이터가 분해 시 확정하여 task.json에 기록하는 파트별 메타데이터(`part_index`, `is_first_part` 등). 호출 어시스턴트가 프롬프트 조립 시 4.2 슬롯에 주입한다.

## 2. 입력 / 출력 규약

- **입력**: 단일 또는 다중 PDF 경로 (`<input>.pdf` …), 또는 **폴더 경로**. 폴더가 주어지면 해당 폴더 내 모든 `*.pdf` 파일을 대상으로 한다.
- **변환 완료 스킵**: 폴더 입력 시 오케스트레이터는 변환 대상 목록을 확정하기 전에 **기존 산출물 존재 여부**를 검사한다. `<원본폴더>_md/<input>.md`가 이미 존재하고 크기 > 0이면 해당 PDF는 **변환 완료(skipped)**로 분류하여 큐에 적재하지 않는다. 스킵된 파일 목록은 사용자에게 보고하고 `agent_report.md`에도 기록한다.
- **최종 산출 경로**: 최종 `.md` 파일은 **원본 PDF가 위치한 폴더의 이름에 `_md` 접미사를 붙인 폴더**에 저장한다. 예: 원본이 `docs/manual/foo.pdf`이면 최종 산출물은 `docs/manual_md/foo.md`. 동일 폴더 내 다중 PDF는 같은 `<원본폴더>_md/`에 모두 저장.
- **작업 루트**: `<workroot>/` (기본값 `./pdf2md_work/`).
- **작업 디렉토리 레이아웃**:

  ```text
  <workroot>/
  ├── queue/
  │   ├── pdf_parts/ ← 오케스트레이터가 qpdf로 사전 생성한 파트별 PDF 추출물
  │   ├── pending/   ← 분해 직후 대기 중인 구간 작업 파일
  │   ├── working/   ← 현재 에이전트가 점유 중인 작업
  │   └── done/      ← 완료된 작업 (병합 입력)
  ├── assets/<input>/ ← 서브에이전트가 pdfimages로 파트별 추출 (partNN-fig-XXX.ext)
  └── out/            ← 병합·린트 스테이징 디렉토리 (최종 산출 위치 아님)

  <원본폴더>_md/<input>.md      ← 최종 병합 산출물 (검증 통과 후 배치)
  <원본폴더>_md/assets/<input>/ ← 최종 이미지 사본
  ```

- **큐 작업 파일**: `<input>__partNN.task.json` 형태. 내용은 입력 PDF 경로, part_source 경로, 페이지 범위, 조건 플래그, 출력 조각 경로, 이미지 디렉토리, 상태.
- **중간 산출물**: `<workroot>/queue/done/<input>__partNN.md` — 병합·검증 완료 후 절차 8에서 `queue/` 전체와 함께 삭제된다.

## 3. 오케스트레이터 작업 범위

### 3.1 핵심 원칙 (오케스트레이터)

- **분할 변환**: 총 페이지 > 50이면 50p 단위로 분할, 구간당 서브에이전트 1개(opus)를 할당한다. 여러 파일을 한 번에 처리할 때도 각 파일을 동일 규칙으로 분해하고 단일 큐로 묶는다.
- **큐 기반 작업 관리**: 작업 단위(= 한 구간)는 `pending → working → done` 디렉토리를 순차 이동. `mv`로 원자 이동하여 점유 경합을 막는다.
- **파일 단위 라운드**: 한 라운드에는 **한 파일의 모든 파트**가 포함되어야 한다. 라운드당 동시 상한은 **20**. 한 파일의 파트 수가 20 이하이면 해당 파일 전체를 1라운드로 처리한다. 파트 수가 20을 초과하는 파일(1,000p 이상)은 20개씩 나눠 복수 라운드로 처리하되, 검증 에이전트 호출은 해당 파일의 모든 라운드가 완료된 후에 한다. 한 라운드에 여러 파일을 섞지 않는다.
- **사전 가공의 허용 범위**: PDF 전처리는 **페이지 분할(`qpdf --pages` 또는 `pdfseparate`+`pdfunite`), 메타데이터 조회(`pdfinfo`)에 한정**한다. `pdftotext`·`pymupdf`·`pdfminer` 등 본문 텍스트 추출 도구를 본문 변환 목적으로 사용하지 않는다.
- **위임 중심 조율**: 오케스트레이터는 변환(호출 어시스턴트), 병합·검증(검증 에이전트)을 위임하고 큐 관리·최종 배치·정리·사용자 보고에 집중한다.

#### 위임 사항

- **호출 어시스턴트**: task.json 목록과 SKILL.md 경로를 전달한다. 프롬프트 조립·서브에이전트 호출·결과 취합은 호출 어시스턴트 책임이다(3.7 참조).
- **검증 에이전트**: 파일별 `done/` 파트 목록, 메타데이터(첨자 플래그, 이미지 수), 원본 PDF 경로, 최종 산출 경로를 전달한다. 병합·린트·검증·오탈자 검사는 검증 에이전트 책임이다(3.8 참조).
- **이미지 추출**: 서브에이전트 책임. 오케스트레이터·검증 에이전트 모두 사전 추출하지 않는다.

### 3.2 절차

1. **입력 스캔 및 스킵 판정**
   - 입력이 폴더이면 `*.pdf` 파일 목록을 수집한다.
   - 각 PDF에 대해 `<원본폴더>_md/<input>.md` 존재 여부를 확인한다(`test -f`, 크기 > 0).
   - 이미 산출물이 존재하는 PDF는 **skipped** 목록에 추가하고 변환 대상에서 제외한다.
   - 스킵 결과를 사용자에게 즉시 보고한다(예: `"3/5 파일 변환 완료 — 스킵: foo.pdf, bar.pdf, baz.pdf / 변환 대상: qux.pdf, quux.pdf"`).
   - 변환 대상이 0개이면 "모든 PDF가 이미 변환됨"을 보고하고 종료한다.

2. **작업 루트 준비**
   - `<workroot>/queue/{pdf_parts,pending,working,done}`, `<workroot>/assets/`, `<workroot>/out/` 생성.
   - 라운드당 동시 상한은 **20**. 한 파일의 파트 수 ≤ 20이면 1라운드로 처리, > 20이면 20개씩 복수 라운드로 분할.

3. **사전 조사** (파일별)
   - `pdfinfo <input>.pdf`로 총 페이지 수 확인.
   - `mkdir -p <workroot>/assets/<input>/` — 서브에이전트가 이미지를 추출할 공용 디렉토리를 사전 생성한다.

4. **분해 · 플래그 확정 · 추출물 생성 · 큐 적재**
   - 파일당 `ceil(total/50)`개 구간을 계산.
   - 각 구간에 대해 **조건 플래그**를 확정한다(3.3 참조).
   - 각 구간마다 **담당 페이지 추출물**을 사전 생성하여 `<workroot>/queue/pdf_parts/<input>__partNN.pdf`에 저장.
     - **권장 (qpdf)**: `qpdf <input>.pdf --pages <input>.pdf <start>-<end> -- <workroot>/queue/pdf_parts/<input>__partNN.pdf`
     - **대안 (pdfseparate + pdfunite)**: `pdfseparate -f <start> -l <end> <input>.pdf /tmp/_p_%d.pdf && pdfunite /tmp/_p_*.pdf <workroot>/queue/pdf_parts/<input>__partNN.pdf && rm /tmp/_p_*.pdf`
     - **절대 금지**: `pdfseparate` 단독 사용(페이지별 파일이 생성되어 단일 추출물이 되지 않음).
     - **참고**: 총 페이지 ≤ 50이고 파트가 1개뿐이면 원본 PDF를 그대로 `cp`하여 `partNN.pdf`로 두어도 무방.
   - 각 구간의 **작업 파일** `<input>__partNN.task.json`을 생성(입력 PDF 경로, part_source 경로, 페이지 범위, 조건 플래그, 출력 조각 경로, 이미지 출력 디렉토리, 상태).
   - 모든 작업 파일을 `queue/pending/`에 배치. 파일별로 그룹화하여 적재한다(평탄 적재 금지).

5. **워커 루프** (파일 단위 라운드)
   - **파일별 순차 처리**: `pending/`에서 파일 1개의 모든 파트를 선택한다. 파트 수 ≤ 20이면 전체를 1라운드로 `working/`에 원자 이동(`mv`). 파트 수 > 20이면 20개씩 분할하여 복수 라운드로 처리한다.
   - 이동한 파트들의 task.json 경로 목록을 **호출 어시스턴트 1개**에 위임한다(3.7 참조).
   - 호출 어시스턴트가 반환한 완료 보고를 수신하여 파일별 메타데이터에 누적(첨자 발견 플래그, 추출 이미지 수 합계). 완료된 작업 파일과 `partNN.md`를 `done/`으로 이동한다.
   - 실패한 작업은 `working/` → `pending/`으로 되돌리고 재시도 카운트를 증가. 임계(기본 2회) 초과 시 `failed/`로 분리 후 보고.
   - 해당 파일의 모든 파트가 `done/`에 모이면 **즉시 절차 6으로 진행**(검증 에이전트 호출). 그 후 다음 파일로 이동한다.

6. **병합·검증 위임** (파일 완료 즉시)
   - 해당 파일의 모든 파트가 `done/`에 모인 시점에 **검증 에이전트 1개**를 호출한다(3.8 참조).
   - 전달 정보: `done/` 내 해당 파일의 파트 목록(`<input>__part01.md` ~ `partNN.md`), 파일별 메타데이터(첨자 발견 플래그, 추출 이미지 수 합계), 원본 PDF 경로, `<workroot>` 경로, 최종 산출 경로(`<원본폴더>_md/<input>.md`).
   - 검증 에이전트가 병합·린트·헤딩 검증·이미지 집계/링크 재작성·오탈자 검사·markdownlint_rules.md 갱신을 수행하고 결과(통과/실패 + 수정 내역 + 보고 항목)를 반환한다.
   - 통과 시 절차 7로 진행. 실패 시 실패 내역을 사용자에게 보고하고 큐를 보존한 채 지시를 기다린다.
   - 절차 7 완료 후 `pending/`에 다음 파일이 남아 있으면 절차 5로 돌아간다.

7. **최종 배치 및 정리**
   - **경로 검증**: 각 입력 PDF에 대해 다음을 모두 확인한다.
     - `<원본PDF폴더>_md/<input>.md` 존재, 크기 > 0.
     - `<원본PDF폴더>_md/assets/<input>/` 이미지 개수가 메타데이터의 `추출 이미지 수` 합계와 일치.
     - 최종 `.md` 내 모든 이미지 링크가 `test -f` 기준으로 실제 파일을 가리킴.
     - `pending/`·`working/`이 비어 있고, 모든 구간이 `done/`에 존재(실패 항목은 `failed/`로 분리되어 보고됨).
   - **정리**: 위 검증이 모두 통과하면 `<workroot>/queue/` 전체를 삭제하고(`rm -rf <workroot>/queue`), `<workroot>/assets/`도 삭제한다.
   - **검증 실패 시 절대 삭제하지 않는다.** 실패 원인을 보고하고 큐를 보존한 채 사용자 지시를 기다린다.
   - **사용자 보고**: 검증 에이전트가 반환한 수정 내역·보고 항목을 사용자에게 요약 보고하고, `agent_report.md`에도 append한다(타임스탬프 + 작업명 헤더).

### 3.3 조건 플래그 확정 규칙

분해(절차 4) 시점에 각 파트에 대해 다음 값을 산출하여 task.json과 프롬프트 슬롯(4.2)에 주입한다.

| 플래그 | 타입 | 산출 방법 |
|---|---|---|
| `part_index` | 문자열 | `01`, `02`, … `NN` (분할 순서, 2자리 zero-pad) |
| `total_parts` | int | `ceil(total_pages / 50)` |
| `is_first_part` | bool | `part_index == "01"` |
| `is_last_part` | bool | `part_index == total_parts` |
| `is_single_part` | bool | `total_parts == 1` |

**사전 플래그가 아닌 것들**:

- **`images_in_range`는 사전 플래그가 아니다.** 서브에이전트가 `part_source`에 `pdfimages -all`을 직접 실행하여 이미지 존재 여부와 개수를 판단하고, PDF를 Read로 시각 확인하면서 본문 내 정확한 위치에 링크를 삽입한다. 추출된 이미지 수는 완료 보고(4.10)에 포함한다.
- **`has_subscripts`는 사전 플래그가 아니다.** 서브에이전트가 `part_source`를 직접 읽어 첨자 발견 여부를 완료 보고(4.10)에 포함하고, 검증 에이전트가 병합 시(3.8) MD033 disable 디렉티브를 주입한다.
- **`boundary_warning`도 사전 플래그가 아니다.** 검증 에이전트가 병합 시(3.8) 파트 경계 줄을 직접 검사하여 이어붙인다. 서브에이전트는 4.5 불변 규칙에 따라 잘린 문장을 임의 완성하지 않는다.

### 3.4 DO / DON'T (오케스트레이터)

#### DO

- 폴더 입력 시 `<원본폴더>_md/<input>.md` 존재 여부를 검사하여 변환 완료 파일을 스킵한다. 스킵 목록은 사용자에게 보고하고 `agent_report.md`에도 기록한다.
- 분해 전 `pdfinfo`로 메타데이터를 수집하여 조건 플래그를 정확히 산출한다.
- 페이지 추출물은 `qpdf --pages`로 단일 파일로 생성한다.
- 서브에이전트가 이미지를 추출할 공용 디렉토리(`<workroot>/assets/<input>/`)를 사전 생성한다.
- 큐 이동(`pending → working → done`)은 `mv`로 원자 처리한다.
- 라운드는 파일 단위로 구성한다. 한 라운드에 여러 파일을 섞지 않는다.
- 라운드당 최대 20 파트. 파트 수 > 20인 파일은 20개씩 복수 라운드로 나눈다.
- 한 파일의 모든 파트가 `done/`에 모이면 즉시 검증 에이전트를 호출하고, 완료 후 다음 파일로 이동한다.
- 서브에이전트 완료 보고의 `첨자 발견` 플래그를 파일별로 누적하여 검증 에이전트에 메타데이터로 전달한다.
- 파일별 모든 파트가 `done/`에 모이면 검증 에이전트를 호출하여 병합·검증을 위임한다.
- 검증 에이전트의 반환 결과(통과/실패, 수정 내역)를 사용자에게 보고하고 `agent_report.md`에 append한다.
- 검증 실패 시 큐를 보존한 채 사용자에게 보고한다.

#### DON'T

- 오케스트레이터가 4절 템플릿을 직접 복사·조립하지 않는다. 프롬프트 조립은 호출 어시스턴트 책임이다.
- 본문을 사전 텍스트화하여 에이전트에 문자열로 넘기지 않는다(`pdftotext`·`pymupdf`·`pdfminer` 등으로 본문 추출 후 전달 금지).
- 오케스트레이터가 이미지를 사전 추출(`pdfimages`)하지 않는다. 이미지 추출은 서브에이전트 책임이다.
- 한 에이전트에 두 개 이상의 작업을 동시에 점유시키지 않는다.
- 오케스트레이터가 병합·린트·오탈자 검사를 직접 수행하지 않는다. 이 책임은 검증 에이전트에 있다.
- 검증 에이전트가 실패를 반환했는데 완료 선언하지 않는다.
- 검증 실패 시 큐를 삭제하지 않는다.

### 3.6 체크리스트 (오케스트레이터)

### 3.7 호출 어시스턴트(Dispatcher)

오케스트레이터로부터 task.json 목록을 받아, 프롬프트 조립·서브에이전트 호출·결과 취합을 전담하는 중간 에이전트.

#### 입력

오케스트레이터가 호출 어시스턴트에 전달하는 정보:

- `working/` 디렉토리에 있는 task.json 파일 경로 목록
- SKILL.md 파일 경로 (4절을 Read하기 위함)

#### 동작

1. SKILL.md의 **4절 서브에이전트 지시문 원본**(4.1~4.10)을 Read 도구로 직접 읽는다.
2. 각 task.json을 읽어 경로·조건 플래그 값을 확인한다.
3. task.json별로 4절 원본을 복사하여 **4.1 변수 정의의 `=` 우변**과 **4.2 조건 플래그 슬롯**만 실제 값으로 치환한다. 4.3~4.10은 축약·생략·재작성 금지.
4. 조립된 프롬프트로 opus **서브에이전트를 병렬 호출**한다(단일 메시지 내 다중 Agent 호출).
5. 서브에이전트 완료 보고(4.10)를 수집하여 **작업별 결과 목록**으로 취합한다.
6. 취합된 결과를 오케스트레이터에 반환한다(파트별 완료 보고 + 성공/실패 상태).

#### DO / DON'T (호출 어시스턴트)

- **DO**: 매 호출 시 4절 원본을 Read로 직접 읽는다(캐시·기억에 의존하지 않는다).
- **DO**: 경로 치환 후 해당 파일이 실제 존재하는지 `test -f`로 확인한다.
- **DON'T**: 4.3~4.10을 축약·생략·재작성하지 않는다.
- **DON'T**: 서브에이전트 결과를 수정·필터링하지 않는다. 있는 그대로 오케스트레이터에 전달한다.
- **DON'T**: 템플릿 외부에 추가 지시사항을 덧붙이지 않는다.

### 3.8 검증 에이전트(Validator)

오케스트레이터로부터 완료된 파트 목록과 메타데이터를 받아, 병합·검증·수정을 수행하고 결과를 반환하는 에이전트.

#### 입력

오케스트레이터가 검증 에이전트에 전달하는 정보:

- 파트 파일 목록: `<workroot>/queue/done/<input>__part01.md` ~ `partNN.md`
- 메타데이터: 첨자 발견 플래그(bool), 추출 이미지 수 합계(int)
- 이미지 소스 디렉토리: `<workroot>/assets/<input>/`
- 원본 PDF 경로: 헤딩 검증 시 참조
- 최종 산출 경로: `<원본폴더>_md/<input>.md`
- 최종 이미지 경로: `<원본폴더>_md/assets/<input>/`

#### 동작

1. **병합**: `part01.md`부터 순서대로 단순 연결 → `<workroot>/out/<input>.md`.
2. **경계 이어붙임**: 각 파트 경계에서 마지막 줄/첫 줄을 검사한다.
   - 문단 단편(문장 중간 끝남) → 한 칸 공백으로 이어붙임.
   - 목록 계속 → 목록 구조 하나로 복원.
   - 표 계속(헤더 없이 시작) → 앞 파트 표에 병합.
   - 내용 자체는 변경하지 않고 구조만 복원.
3. **첨자 디렉티브 주입**: 첨자 발견 플래그가 true이면 병합 파일 최상단에 `<!-- markdownlint-disable MD033 -->` 1회 삽입.
4. **이미지 집계**: `<workroot>/assets/<input>/`의 이미지를 `<원본폴더>_md/assets/<input>/`으로 복사. 대상 디렉토리 없으면 `mkdir -p`.
5. **이미지 링크 재작성**: `../../assets/<input>/...` → `assets/<input>/...`로 일괄 재작성.
6. **markdownlint 실행**: `markdownlint <workroot>/out/<input>.md`. 위반 시 구조·포맷 한정 자가 수정 후 재검증. 원문 의미 변경이 필요한 위반은 실패로 보고.
7. **헤딩 순서·계층·표기 검증**: 원문 PDF를 Read하여 목차/장·절 번호와 최종 `.md` 헤딩을 대조. 순서·계층·표기 불일치 시 자가 수정(구조 한정) 또는 실패 보고.
8. **이미지 링크 해소 검증**: 모든 이미지 링크의 실제 파일 존재 여부를 `test -f`로 확인. 이미지 0개인 경우에도 명시적으로 "0/0 통과" 기록.
9. **오탈자 검사**: `language_tool_python`으로 en-US/ko-KR 검출. 전처리(수식·HTML 주석 제거) 후 실행. 자동 수정은 단일 후보 + TYPOS/MISSPELLING 카테고리 한정, BRITISH_ENGLISH_DETECTOR 제외. 다중 후보·문법은 미수정 보고. 리포트 생성 후 `python3 scripts/filter_typo_report.py`로 FP 제거.
10. **markdownlint_rules.md 갱신**: 위반 규칙을 `.claude/skills/pdf2md/markdownlint_rules.md`와 대조하여 미등록 규칙 append, 차이가 큰 경우 `## 사용자 협의 필요` 섹션에 기록. SKILL.md 본문에 재이식 금지.
11. **최종 배치**: 린트 통과 후 `<workroot>/out/<input>.md`를 최종 경로 `<원본폴더>_md/<input>.md`로 복사.

#### 반환

```text
검증 결과:
- 파일:           <input>.md
- 상태:           <통과 | 실패>
- markdownlint:   <위반 N건 → 자가 수정 N건, 미해소 N건>
- 헤딩 검증:      <통과 | 불일치 N건 (상세)>
- 이미지 링크:    <N/M 해소 (이미지 없음 시 "0/0 통과")>
- 오탈자:         <자동 수정 N건, 미수정 M건 (보고 항목)>
- rules.md 갱신:  <신규 N건 append | 변경 없음 | 사용자 협의 필요>
- 수정 내역:      <줄·규칙ID·수정 요지 목록>
```

#### DO / DON'T (검증 에이전트)

- **DO**: 원문 PDF를 Read하여 헤딩 대조 시 참조한다.
- **DO**: markdownlint 설정이 프로젝트에 `.markdownlint.json`으로 있으면 그것을 우선한다.
- **DO**: 자동 수정·미수정 항목 모두 반환에 포함한다.
- **DON'T**: 원문 텍스트·의미를 변경하지 않는다. 수정 범위는 구조·포맷에 한정.
- **DON'T**: 다중 후보·문맥 의존·문법 카테고리 오탈자를 임의 자동 수정하지 않는다.
- **DON'T**: markdownlint_rules.md의 기존 문구를 임의 수정하지 않는다.
- **DON'T**: 페이지 단위·경계 마크를 삽입하지 않는다.

## 4. 서브에이전트 지시문 원본

> **호출 어시스턴트(Dispatcher)가 사용하는 섹션**: 호출 어시스턴트는 이 섹션 전체(4.1~4.10)를 Read 도구로 직접 읽어 프롬프트를 조립한다. 4.1 변수 정의의 `=` 우변과 4.2 조건 플래그 슬롯만 task.json 값으로 치환하며, 4.3~4.10은 축약·생략·재작성을 금지한다(3.7 참조). 오케스트레이터는 이 섹션을 직접 복사하지 않는다.

### 4.1 역할 / 입력

역할: PDF 구간을 무손실 마크다운으로 변환하는 전문 에이전트. 이미지 추출·위치 매칭·링크 삽입까지 단독 수행한다.

part_source  = <workroot>/queue/pdf_parts/<input>__partNN.pdf
page_range   = <start>-<end>
image_dir    = <workroot>/assets/<input>/
image_prefix = partNN-fig
output_file  = <workroot>/queue/working/<input>__partNN.md


### 4.2 작업 조건 (조건 플래그)

```text
작업 조건 (오케스트레이터가 분해 시 확정):
- part_index:      <01 | NN>
- total_parts:     <N>
- is_first_part:   <true | false>
- is_last_part:    <true | false>
- is_single_part:  <true | false>
```

### 4.3 핵심 원칙

- **무손실 보존**: 원문 텍스트는 삭제·요약·의역·"명확화" 금지. 문단·목록·표의 순서를 그대로 유지한다.
- **구조화**: 제목 계층(`#`~`######`), 목록, 표, 코드/수식 블록을 마크다운 문법으로 복원한다.
- **단일 책임**: 담당 페이지 범위만 변환한다. 타 구간 산출물을 참조·수정하지 않는다.
- **직접 읽기**: `part_source` PDF만 Read 도구로 직접 읽어 변환한다. 다른 입력을 참조하지 않는다.
- **이미지 추출·매칭 책임**: 서브에이전트가 `pdfimages -all`로 담당 `part_source`의 이미지를 직접 추출하고, PDF를 시각 확인하여 본문 내 정확한 위치에 링크를 삽입한다. 본문에 참조가 없는 추출 이미지(로고, 장식 등)는 orphan으로 판정한다.

### 4.4 변환 절차

1. `part_source` PDF를 Read 도구로 연다.
2. 문서 구조를 파악한다(제목 계층, 섹션 번호, 표·수식·그림 배치, 캡션, 머리말/꼬리말, 페이지 번호). 이때 **그림이 등장하는 위치와 캡션을 메모**한다.
3. **이미지 추출**: `pdfimages -all <part_source> <workroot>/assets/<input>/partNN-fig`을 Bash로 실행한다. 추출 결과 파일 목록을 `ls <workroot>/assets/<input>/partNN-fig*`로 확인한다.
4. **이미지 병합·위치 매칭·description 작성**:
   - **병합 판정**: PDF를 Read로 시각 확인한 그림 1개에 대해 `pdfimages` 추출 파일이 여러 개(레이어·조각·오버레이 등)로 분리된 경우, `magick composite` 또는 `magick convert +append`/`-append`로 단일 이미지로 병합한다. 병합 결과 파일명은 `partNN-fig-XXX-merged.ext`로 저장하고, 원본 조각은 삭제한다.
   - **위치 매칭**: Read로 확인한 PDF 내 그림 등장 순서와 추출(또는 병합 후) 이미지를 대응시킨다.
   - **description 작성**: 각 이미지에 대해 앞뒤 본문 문맥과 캡션을 참고하여 이미지 내용을 설명하는 alt 텍스트(description)를 작성한다.
   - 본문에 그림 참조가 없는 이미지(로고, 장식, 배경 등)는 **orphan**으로 판정하여 링크하지 않되, 완료 보고에 orphan 수를 명시한다.
5. 4.2의 조건 플래그를 확인하고 4.6의 분기 규칙 중 적용 대상을 식별한다.
6. 4.7에 따라 `.claude/skills/pdf2md/markdownlint_rules.md`을 Grep으로 조회하여 "변환 시점에 피할 수 있는 규칙" 목록을 확보한다.
7. 원문 순서를 유지하며 4.5 불변 규칙과 4.6 분기 규칙에 따라 마크다운으로 변환한다. 그림이 등장하는 정확한 위치에 4.5의 이미지 링크 규약으로 삽입한다.
8. 산출물을 `<workroot>/queue/working/<input>__partNN.md`에 저장한다.
9. 4.9 자가 체크리스트로 점검 후 4.10 형식으로 완료 보고한다.

### 4.5 불변 변환 규칙

1. **직독 전용 + 이미지 추출**: `part_source` PDF만 Read 도구로 직접 읽는다. `pdftotext`·`pdfminer`·`pymupdf(fitz)`·`pypdf`·`pdfplumber` 등 **텍스트** 추출 도구·패키지는 어떤 용도로도 사용하지 않는다. 원본 전체 PDF나 타 구간 PDF도 읽지 않는다. **예외**: `pdfimages -all`은 이미지 추출 용도로만 허용한다(본문 텍스트 추출 아님).
2. **원문 보존**: 원문 텍스트를 삭제·요약·의역·"명확화"하지 않는다. 원문 순서를 유지한다.
3. **헤딩 원문 일치**: 원문 장·절 번호 체계(예: `1`, `1.1`, `1.1.1`)와 제목 텍스트를 그대로 옮긴다. 원문의 장·절·하위절 깊이에 맞춰 `#`~`######` 레벨을 부여하고, 한 단계 이상 건너뛰지 않는다(예: `##` 다음 바로 `####` 금지). 임의 번호 재부여·번역·축약 금지.
4. **첨자·위첨자**: 아래 첨자는 `<sub>...</sub>`, 위 첨자는 `<sup>...</sup>`로 표기한다. 예: `R<sub>eH</sub>`, `H<sub>2</sub>O`, `x<sup>2</sup>`, `σ<sub>max</sub>`. 원문 의미 보존을 위해 생략 금지. **`<!-- markdownlint-disable MD033 -->` 디렉티브는 에이전트가 찍지 않는다**(오케스트레이터가 병합 후 1회 주입).
5. **표**: 원문 구조·정렬·헤더·셀 내용을 최대한 보존한다. 셀 병합은 마크다운 한계 내에서 주석 또는 분리로 표현한다.
6. **코드 블록**: 펜스 코드로 감싸고 언어를 명시한다(`python`, `bash` 등, 알 수 없으면 `text`).
7. **수식**: 인라인은 `$...$`, 블록은 `$$...$$` (LaTeX). 원문이 수식 이미지라면 이미지 링크로 대체하고 캡션을 유지한다.
8. **제거 대상**: 본문 흐름과 무관한 **페이지 번호, 반복 머리말/꼬리말**은 제거한다.
9. **페이지 경계 마크 금지**: `--- page N ---` 등 페이지 단위·경계 구분 마크를 어떤 형태로도 삽입하지 않는다.
10. **경계 보존**: 담당 범위의 첫/마지막 문단·목록·표가 경계에서 잘려 보여도 **임의로 문장을 완성하거나 마침표·단어를 추가하지 않는다**. 원문 그대로 기록한다. (오케스트레이터가 병합 시 이어붙인다.)
11. **이미지 링크 규약**: 그림은 `![description](../../assets/<input>/partNN-fig-XXX.ext)` 형태의 상대경로로 삽입한다(`queue/working/` 기준). `partNN-fig-XXX.ext`는 4.4 단계 3에서 `pdfimages`가 생성한 실제 파일명을 사용한다. alt 텍스트(description)는 다음 우선순위로 결정한다: **(1)** 원본 캡션이 있으면 그대로 사용, **(2)** 캡션이 없으면 앞뒤 본문 문맥을 참고하여 이미지가 나타내는 내용을 한 문장으로 기술한다(예: `![Table 3에서 참조하는 응력-변형률 곡선](...)`, `![절차 2.3의 검사 흐름도](...)`). 추측이 어려운 경우 `![Image](...)`로 남긴다. (병합 후 오케스트레이터가 최종 위치 기준으로 경로를 재작성한다.)
12. **분할 추출 이미지 병합**: `pdfimages`가 PDF 상의 단일 그림을 여러 파일(레이어·조각·텍스트 오버레이 등)로 분리 추출한 경우, `magick composite` 또는 `magick convert +append`/`-append`로 병합하여 단일 파일로 만든다. 병합 결과는 `partNN-fig-XXX-merged.ext`로 저장하고 원본 조각은 삭제한다.
13. **인라인 이미지 금지**: 인라인 base64 이미지 임베드 금지. 반드시 파일 링크.

### 4.6 조건부 분기 규칙

4.2의 조건 플래그 값에 따라 다음과 같이 분기한다.

- **`is_first_part=true`**:
  - 파일 최상단에 문서 제목을 H1(`# 제목`)으로 작성한다.
  - 원문에 목차(TOC)가 담당 범위 내 있다면 그대로 변환한다.

- **`is_first_part=false`**:
  - **H1 작성 금지**. 담당 범위 최상위 섹션부터 원문 계층에 맞는 레벨(`##` 또는 그 이하)로 시작한다.
  - 문서 제목을 반복하지 않는다.

- **`is_last_part=true`**:
  - 원문의 부록·색인·개정이력이 담당 범위에 있으면 포함한다.

- **`is_single_part=true`**:
  - 문서 전체를 단독 변환하는 경우이며, `is_first_part=true` + `is_last_part=true`의 규칙이 동시에 적용된다.

- **이미지 추출 결과 0개**:
  - `pdfimages -all` 실행 후 추출 파일이 0개이면 이미지 링크 규칙(4.5-11)은 적용 대상이 없다. 억지 삽입 금지.

- **이미지 추출 결과 1개 이상**:
  - PDF를 Read로 시각 확인한 그림 등장 위치에만 링크한다. 본문에 참조가 없는 추출 이미지(로고, 장식 등)는 orphan으로 판정하여 링크하지 않고, 완료 보고에 orphan 수를 명시한다.

### 4.7 markdownlint 주의점 (외부 규칙 파일 참조)

markdownlint 규칙 가이드는 상시 갱신되는 외부 파일에서 관리한다. 본 섹션에 규칙을 이식하지 않는다.

- **규칙 파일 경로**: `.claude/skills/pdf2md/markdownlint_rules.md` (프로젝트 루트 기준 상대경로)
- **조회 방법**: 변환 시작 전 Grep 도구로 규칙 목록을 확보한다.
  - 권장 명령: `Grep pattern='^- \*\*MD' path='.claude/skills/pdf2md/markdownlint_rules.md' output_mode='content'`
  - 특정 규칙 상세가 필요하면 해당 라인을 `-A 1` 옵션으로 재조회하거나 동일 경로를 Read로 직접 연다.
- **적용 범위**: 파일 내 "변환 시점에 피할 수 있는 규칙" 섹션의 항목만 서브에이전트가 사전 회피한다. "병합 후 검증 대상 규칙" 섹션은 오케스트레이터 책임이므로 서브에이전트는 참고만 한다.
- **결과 관리**: `markdownlint` CLI 실행·위반 수집·자가 수정·재검증·사용자 보고는 **모두 오케스트레이터 책임**이다. 서브에이전트는 변환 중 피할 수 있는 위반을 선제적으로 피할 뿐, `markdownlint`를 실행하지 않는다.

### 4.8 DO / DON'T (서브에이전트)

#### DO (서브에이전트)

- `part_source` PDF를 Read 도구로 **직접** 읽어 시각 구조(제목, 표, 수식, 그림 배치)를 파악한다.
- `pdfimages -all <part_source> <workroot>/assets/<input>/partNN-fig`으로 이미지를 추출하고, `ls`로 추출 결과를 확인한다.
- `pdfimages`가 단일 그림을 여러 파일로 분리 추출한 경우, `magick`으로 병합하여 단일 이미지로 만든 뒤 링크한다.
- PDF Read 시 확인한 그림 등장 순서와 추출(또는 병합 후) 이미지를 대응시켜 정확한 위치에 이미지 링크를 삽입한다.
- 본문에 참조가 없는 추출 이미지(로고, 장식 등)는 orphan으로 판정하여 링크하지 않되, 완료 보고에 orphan 수를 명시한다.
- 이미지 alt 텍스트는 **(1)** 원본 캡션이 있으면 그대로, **(2)** 없으면 앞뒤 문맥을 참고하여 이미지 내용을 한 문장으로 기술한다.
- 표·코드 블록의 언어·정렬 정보를 최대한 보존한다.
- 첨자·위첨자·수식 변수의 의미를 잃지 않도록 `<sub>`/`<sup>`/LaTeX 태그를 사용한다.
- 완료 보고에 추출 이미지 수, 삽입 이미지 수, orphan 수, 첨자 발견 여부, 경계 잘림 관찰, 특이사항을 포함한다.
- `is_first_part`에 따라 H1 처리를 분기한다.

#### DON'T (서브에이전트)

- `pdftotext`, `pdfminer`, `pymupdf(fitz)`, `pypdf`, `pdfplumber` 등 **텍스트** 추출 도구·패키지를 사용하지 않는다(`pdfimages`는 이미지 추출 전용으로 허용).
- 원본 전체 PDF나 타 구간 PDF를 Read하지 않는다. `part_source`만 읽는다.
- 원문을 축약·의역·"명확화"하지 않는다.
- 페이지 경계 마크(`--- page N ---` 등)를 삽입하지 않는다.
- 담당 범위 경계에서 잘린 문장을 임의로 완성하지 않는다.
- 인라인 base64 이미지를 임베드하지 않는다.
- 타 구간 산출물을 읽거나 수정하지 않는다.
- `is_first_part=false`인데 H1을 찍지 않는다.
- `<!-- markdownlint-disable MD033 -->` 디렉티브를 직접 찍지 않는다(오케스트레이터 담당).
