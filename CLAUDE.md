## 오케스트레이터/서브에이전트
- **병렬 호출은 허용**하되, 서브에이전트가 오케스트레이터로 반환하는 응답은 서로 혼합되지 않아야 한다.
- **이유**: 여러 서브에이전트의 응답이 동시에 도착하면 어느 결과가 어느 작업의 산출물인지 구분·추적이 어려워진다.
- **응답 분리 방법**:
  - 각 서브에이전트가 자기 결과를 **고유 파일**(작업ID·에이전트명 포함)로 기록하고, 오케스트레이터에는 파일 경로·요약만 반환한다.
  - 오케스트레이터는 반환된 결과를 **에이전트 단위로 분리해서 수신·검증**한 뒤 병합한다 (여러 응답을 한 번에 섞어서 처리 금지).

## 영문/국문 번역 요청시
- 번역 시 멀티에이전트를 무조건 사용한다. 별다른 요청이 없으면 50줄당 1개 에이전트를 할당한다. 
- 번역결과를 하나의 에이전트가 취합한 후 1개 파일로 합치지 말고 동일한 line에 바로 작성한다.
- 원문파일은 나두고, 접미사_k를 추가한다. 

## 국문 파일 수정 시
- 동일한 이름(뒤에 _ko)에 서브에이전트로 영문도 수정

## pdf, html, json, md 등 파일 형식 변환 시
- 에이전트가 본문을 직접읽고 변환, 원본 텍스트 유지, 구조 유지 필수

## llm 에서 요청한 질의는 모두 저장
- user_query.md 에 질의 내용 자동 저장 (UserPromptSubmit hook → .claude/log_query.py)

## 스킬별 가상환경
- 각 스킬이 Python 패키지를 필요로 하면 스킬 디렉토리 내 `.venv/`에 가상환경을 생성·사용한다.
- 경로: `.claude/skills/<skill-name>/.venv/`
- 패키지 설치·스크립트 실행 시 반드시 해당 `.venv`의 pip/python을 사용한다.
- `.venv/`는 `.gitignore`에 등록되어 있으므로 커밋하지 않는다.

## 에이전트 처리 보고
에이전트는 작업 수행 중 요청 사항이 모호하거나 구체성이 부족하다고 판단되는 경우, 해당 내용을 사용자에게 반드시 보고해야 한다.
보고 내용은 요약하여 agent_report.md 에 append 한다 (타임스탬프 + 작업명 헤더 포함).

보고에는 다음 내용을 포함한다.
- 모호하거나 정보가 부족한 요청 내용
- 해당 내용을 에이전트가 어떻게 해석했는지
- 해석에 따라 실제로 어떻게 처리했는지
- 처리 과정에서 발생했거나 발생 가능했던 문제점
- 각 항목의 심각도(상 / 중 / 하)

## 공유하는 경로, 파일명 등
- 번호 ↔ 텍스트 일관성 검증: 번호와 식별자(파일명/섹션명/단계명 등)를 함께 쓸 때 두 표현이 항상 같은 대상을 가리키는지 확인
- 단일 정본(SSOT) 원칙: 경로·파일명 같은 식별자는 한 곳에서 정의·관리하고, 다른 문서는 정본을 참조하기만 함 (중복 정의 금지)

### SSOT 정본

- **파일/경로**: 정본 `shared/files_registry.yaml`. 본문 참조는 `**[A05]** \`filename\`` (ID + 백틱 파일명). ID: `A##` artifacts / `P##` paths / `D##` docs.
- **임계값**: 정본 `shared/thresholds.yaml`. 본문 참조는 `{{key:value}}` (값 인라인, `scripts/update_thresholds.py`로 동기화).
- **검증**: `python3 scripts/validate_files.py` (등록 여부 + ID↔파일명 일치 + 임계값 키 확인)
